OSI Model

The Seven Layers (Top → Bottom)

  1. Application
    • Application-specific protocols and services.
  2. Presentation
    • Data representation and translation (e.g., encoding formats).
  3. Session
    • Connection management, dialog control (request/response patterns; RPC concept).
  4. Transport
    • Process-to-process communication.
    • Reliability, flow control, congestion control (e.g., TCP).
  5. Network
    • Host-to-host packet delivery.
    • Forwarding + routing (e.g., IP).
  6. Data Link
    • Framing, MAC addressing, error detection/correction on a link.
    • Medium access (e.g., CSMA/CD in Ethernet).
  7. Physical
    • Transmission of raw bits over the medium.

What you should be able to do

  • Identify which layer provides a given functionality (e.g., reliability → Transport).
  • Explain why some functions must be end-to-end (e.g., reliability) while others are hop-by-hop (e.g., framing).

Switching and Store-and-Forward

Store-and-Forward Transmission

  • A switch/router receives the entire packet (or frame) before forwarding it.
  • This is characteristic of packet switching (not circuit switching).

Packet Switching vs Circuit Switching

  • Circuit switching: reserves resources end-to-end; consistent latency; inefficient when idle.
  • Packet switching: statistical multiplexing; efficient; variable delay/queueing.

Reliable Data Transfer (RDT)

This section covers constraints and reasoning patterns used in Selective Repeat and Go-Back-N.

Selective Repeat (SR)

Key ambiguity constraint: sequence number bits vs window size

Let:

  • N = sequence number space size = 2^k for k bits
  • W = sender window size (and receiver window size in SR)

Constraint to avoid ambiguity:

  • N >= 2W (often stated as W <= N/2)

Why: after wraparound, an old packet could be mistaken for a new one unless the window is small enough relative to the sequence space.

What you should be able to do

  • Given W, compute minimum k such that 2^k >= 2W.
  • Explain ambiguity with wraparound using a short example.

Go-Back-N (GBN)

Properties

  • Sender maintains a window of unACKed packets.
  • Receiver only accepts in-order packets (typically discards out-of-order).
  • ACKs are cumulative: ACK k means “received everything up to k (or next expected = k+1 depending on convention).”

Typical exam reasoning patterns

  • Identify the receiver’s next expected sequence number k.
  • If medium does not reorder:
    • Packets that have been sent but not ACKed form a contiguous range.
    • Sender window consists of consecutive sequence numbers.

What you should be able to do

  • List all possible sender-window contents given receiver expectation k and window size W.
  • Determine possible ACK values in flight back to the sender given the constraints.

Forwarding and Routing Tables (Datagrams vs VC)

Datagram Forwarding (IP-style)

  • Each packet is routed independently.
  • Switch/router uses a routing table mapping destination prefixes (or destinations) to outgoing interfaces.
  • No per-flow state required.

What you should be able to do

  • Given a network diagram with interface labels, build routing tables that map destinations to correct next hops/interfaces.

Virtual Circuit (VC) Forwarding

  • A VC is established along a path before data transfer.
  • Each switch keeps a VC table mapping:
    • (incoming interface, incoming VCI) → (outgoing interface, outgoing VCI)

VCI allocation rule (typical in problems)

  • “Always use the smallest available VCI.”
  • VCIs are local identifiers: the number can change hop-to-hop.

What you should be able to do

  • For a sequence of VC setup requests, update per-switch VC tables correctly.
  • Track VCI availability per interface.

Core tradeoff summary

  • Datagram: flexible, no setup, robust to failure; but routing decision per packet.
  • VC: potentially faster forwarding and consistent path; but setup overhead and failure complexity.

End-to-End Delay on Multi-Hop Paths

Delay components

  • Transmission delay: p / s
    where p = packet size (bits), s = link rate (bps)
  • Propagation delay: d (seconds)
  • Processing and queueing delay: often assumed zero in exam setups unless stated.

Store-and-forward multi-hop behavior

On a path A → B → C → D:

  • Each hop requires transmission + propagation.
  • Bottleneck link (slowest rate) dominates spacing between successive packets.

Back-to-back packet timing pattern

If A sends 2 packets back-to-back:

  • The start time of packet 2 is typically delayed by:
    • transmission time of packet 1 on the first link + propagation (depending on how the problem defines “back-to-back” in time).

What you should be able to do

  • Compute arrival time of the last bit of the second packet at the destination.
  • Identify where packet spacing increases (at slower links) and where it can compress (at faster links).

Error Detection: CRC

CRC basics

Given:

  • Data bits D
  • Generator polynomial G(x) of degree r

Steps:

  1. Append r zeros to D → D * 2^r
  2. Perform modulo-2 division of D * 2^r by G
  3. Remainder R has at most r bits
  4. Transmit codeword: C = D || R

Modulo-2 arithmetic reminders

  • Addition/subtraction are XOR
  • No carries/borrows

Undetectable error patterns

An error pattern E is undetectable if:

  • E is divisible by G (i.e., E mod G = 0)
  • Equivalent: if the corrupted codeword is still a multiple of G

What you should be able to do

  • Encode a given dataword with a given generator polynomial.
  • Construct (or describe) an undetectable error pattern as a multiple of G.

Error Detection: Internet Checksum

Internet checksum (1’s complement)

  • Split data into 16-bit words.
  • Add words using 1’s complement addition:
    • normal binary addition
    • if there is an overflow carry out of MSB, wrap it around and add to LSB (“end-around carry”)
  • Final checksum = bitwise NOT of the final sum.

What you should be able to do

  • Perform 16-bit 1’s complement addition correctly.
  • Handle multiple carries.
  • Produce final complemented checksum.

Ethernet (CSMA/CD): Collision + Timing

Ethernet CSMA/CD essentials

  • Carrier Sense: listen before transmitting.
  • Multiple Access: many nodes share the medium.
  • Collision Detection: detect collision while transmitting on wired Ethernet.

Collision timing concepts

  • A collision must be detected while still transmitting; otherwise a sender might finish and falsely assume success.
  • Ethernet uses:
    • 64-bit preamble
    • 32-bit jam sequence after collision detection (simplified in problem statements)

Minimum frame size rationale

  • Ensures transmitter is still sending when a collision from the far end can propagate back.
  • Rough idea: minimum transmission time must be ≥ round-trip propagation time.

What you should be able to do

  • Convert between:
    • propagation time (seconds) ↔ bit times (bit durations)
  • Determine:
    • whether A detects collision before finishing its frame
    • when each host finishes sending jam sequence

Line Encoding: NRZ, Manchester, NRZI

NRZ (Non-Return-to-Zero)

  • Given mapping: 1 = High, 0 = Low
  • No guaranteed transitions → poor clock recovery with long runs.

Manchester

  • Guaranteed transition in middle of every bit period.
  • Typically:
    • 1 and 0 differ by the direction of mid-bit transition (convention may vary).
  • Good for synchronization; costs more bandwidth.

NRZI

  • Start at a known initial level (problem may specify start low).
  • Common convention:
    • 1 → transition
    • 0 → no transition

What you should be able to do

  • Draw waveforms for a given bit pattern under each encoding scheme.
  • Explain tradeoffs (clock recovery vs bandwidth).

IP Addressing vs MAC Addressing

IPv4 address properties

  • 32-bit logical address.
  • Hierarchical structure enables aggregation and scalable routing.

MAC address properties

  • Typically 48-bit (6-byte) link-layer address.
  • Flat (not hierarchical) from a routing standpoint.

If IP used MAC addresses instead (conceptual redesign)

Potential advantage

  • Unification of identity; avoid separate logical/physical identifiers in simple setups.

Strong disadvantages (high-yield)

  • Routing scalability collapse: flat addresses break aggregation; routing tables grow massively.
  • Mobility and topology changes: logical addressing tied to topology is valuable.
  • Administrative control: IP addressing plans support structured network management.

What you should be able to do

  • Argue the hierarchy/aggregation point clearly and concretely.

IPv4 Fragmentation and Reassembly

Why reassembly at endpoints (not routers)?

Two strong reasons:

  1. Performance + state: routers would need to buffer fragments, track per-datagram state, and wait for all fragments.
  2. Loss handling: if any fragment is lost, reassembly fails; doing this in routers wastes resources and increases complexity.

Fragmentation rules

  • Fragment offset is in 8-byte units.
  • Each fragment carries its own IP header.
  • Total length fields and offsets must be consistent.

MSS relationship

  • Often used in TCP questions:
    • MSS = MTU - IP_header - TCP_header
  • Application data passed to TCP may become multiple TCP segments, then possibly fragmented by IP on smaller-MTU links.

What you should be able to do

  • Given MTUs on links, sketch packets per link:
    • size of each IP fragment
    • offsets
    • header overhead
  • Compute how many total bits are delivered to the network layer at the destination.

IP vs MAC When Moving Networks

MAC address

  • Usually does not change when moving between departments (it is tied to the NIC).
  • Exceptions exist (virtual NICs, replacement hardware), but standard assumption: MAC remains fixed.

IP address

  • Often must change if you move to a different subnet/VLAN, because IP is tied to topology and routing domains.
  • If the new department is within the same IP subnet and configuration, it might not change—but typical exam assumption: different department → different subnet → IP changes.

What you should be able to do

  • State assumptions explicitly:
    • Are departments different subnets?
    • Is DHCP used?
    • Is L2 extended across departments?

Subnetting Essentials

Key definitions

  • Subnet mask splits IP into:
    • network/subnet portion
    • host portion
  • Subnet number: IP AND subnet mask
  • CIDR notation: /n indicates the number of leading 1s in the subnet mask.

Skills

  • Convert between:
    • dotted decimal mask (e.g., 255.255.255.128) and /n (e.g., /25)
  • Determine:
    • which subnet an IP belongs to
    • routing table entries per subnet

Common masks (memorize)

  • /24 = 255.255.255.0
  • /25 = 255.255.255.128
  • /23 = 255.255.254.0

Routing with Subnets + Longest Prefix Match

Longest Prefix Match

  • When multiple entries match a destination, pick the entry with the most specific prefix (largest /n).

Default route

  • Used when no more-specific entry matches.
  • Written as <default> or 0.0.0.0/0.

What you should be able to do

  • Given a routing table with multiple subnets:
    • determine which interface/next-hop is used for each destination
  • Explain why a more-specific subnet entry overrides a broader one.

Link-State Routing (Dijkstra)

Conceptual steps

  1. Each node learns the full topology (link states + costs).
  2. Run Dijkstra from the source node:
    • Maintain:
      • confirmed set (shortest path finalized)
      • tentative distances for others
  3. Produce:
    • shortest-path tree
    • next-hop routing table

What you should be able to do

  • Perform Dijkstra step-by-step.
  • Track:
    • current minimum tentative node
    • updated tentative costs
    • predecessor pointers
  • Derive next-hop decisions from the shortest paths.

Distance Vector Routing (Bellman-Ford)

Core update equation

For node x computing distance to destination y:

  • D_x(y) = min over neighbors v { c(x,v) + D_v(y) }

Convergence concept

  • Nodes repeatedly exchange vectors with neighbors until stable.

Typical exam tasks

  • Given a topology and link costs:
    • compute final distances from a node X to all nodes
    • sometimes identify next-hop choices (via which neighbor gives min)

Failure modes to avoid

  • Mixing up cost to neighbor and neighbor’s distance to destination
  • Forgetting to recompute minima after updates

Quick Concept Questions

Why CSMA/CD cannot be used in wireless

  • A wireless sender typically cannot reliably detect a collision while transmitting:
    • self-interference dominates
    • hidden terminal problem: collisions happen at receiver without sender awareness

Drawback of virtual-circuit-based forwarding

  • Requires per-flow state in network.
  • Setup overhead and management complexity.
  • Failures require teardown/rebuild; less flexible than datagrams.

Why Ethernet uses binary exponential backoff

  • Reduces probability of repeated collisions under high contention by expanding the random wait window after each collision.
  • Stabilizes throughput as load increases.

Why Ethernet has a minimum packet size

  • Ensures collision detection is possible: transmitter must still be sending when the collision signal can propagate back from the far end of the network.
